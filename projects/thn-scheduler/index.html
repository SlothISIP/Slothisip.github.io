Overview

I designed and built THN AI Production Scheduler as a solo developer handling both backend and frontend end-to-end, with guidance from my team manager. This intelligent production planning system serves wire harness manufacturing. I implemented optimization for 67,738 circuits across 21 machine groups using 15 different algorithms, achieving 22% improvement in load balance (CV: 0.45→0.35) and 32% reduction in applicator changes (APP: 178→120).

Note: This project was developed for industrial clients under NDA. Specific client names and proprietary datasets have been anonymized. The methodology and architecture described here accurately represent my contributions.

My Contributions

  • Algorithm Development: I implemented and benchmarked 15 scheduling algorithms including ALNS (Adaptive Large Neighborhood Search), CP-SAT, and custom Two-Phase Balanced approaches
  • Optimization Engine: I designed the multi-objective optimization framework minimizing weighted sum of CV (load balance), APP (applicator changes), and makespan
  • Backend Development: I built the complete FastAPI backend with real-time schedule generation and algorithm comparison APIs
  • Frontend Development: I developed the Streamlit dashboard with interactive Gantt charts, 3D factory layout visualization, and convergence analysis plots
  • Production Deployment: I deployed the system optimizing production schedules for 67,738 circuits with sub-500ms response time for real-time re-scheduling

Key Technical Decisions

Decision ALNS for Real-time, CP-SAT for Batch
Context Production floor requires sub-second re-scheduling when machines go down; overnight planning can use exact solvers
Options Single algorithm vs. Algorithm selection based on time budget
Rationale ALNS achieves 85% of optimal in 500ms for real-time re-scheduling; CP-SAT finds near-optimal solution given 90-minute overnight compute budget
Decision Multi-Objective Weighted Sum over Pareto Frontier
Context Three competing objectives: load balance (CV), setup time (APP), makespan
Options Pareto frontier exploration, Weighted sum, Lexicographic ordering
Rationale Production managers prefer single "best" schedule over choosing from Pareto set; weighted sum with factory-calibrated weights (CV: 0.4, APP: 0.4, Makespan: 0.2) aligned with business priorities
